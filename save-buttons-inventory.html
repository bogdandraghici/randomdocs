Caveats

  • Entries were found via primary-button markup (mat-flat-button color="primary", flx-button default/primary, bg-primary / bg-info-*). Visual rendering depends on the active theme; in the default FlowX theme these resolve to a blue background with light text.
  • Generic confirmation modals styled as primary (e.g., Confirm, Paste) are listed. Pure destructive confirms like "Delete" buttons styled as primary were excluded from the main lists — ping if you want them too.
  • A "Save" label alone doesn't always mean it persists to the backend — some modals just commit changes back into the parent form's in-memory state.
  • Paths are repo-relative; file paths use :LINE suffix for direct navigation in most IDEs.
